# Hum Pitch Studio Web

这是网页版哼唱转 MIDI 工具。它在浏览器里完成录音、编辑、和弦、试听和 MIDI 导出，并优先调用本机 Basic Pitch / ONNX 高精度识别引擎，再叠加单声部 pYIN 音高轨迹校准。

界面现在拆成四个工作页：录音/导入、识别修音、和弦编配、试听导出。录音或导入音频后会自动进入识别修音页，顶部导航也可以随时横向切换，不会丢失已经载入的音频、识别音符或和弦编辑结果。

新版把本地高精度识别接成 `/api/analyze` 服务；如果本地服务不可用，会退回 Web Worker 里的备份识别引擎，生成时页面不会卡住。录音改为直接采集 PCM，减少浏览器压缩带来的瞬态误差。哼唱音高会用 Basic Pitch 切音，再用 pYIN 在每个音的中后段稳定区域重新确认音高，减少音头、滑音和尾音造成的半音碎片。

稳定性优化包括：音频预滤波、动态噪声门、YIN + 自相关联合音高检测、音高轨迹平滑、Space 手动分音的短窗口采样，以及识别调性后的错音/八度误判修正。开启 Space 手动分音时，每次 Space 都对应一个音符槽位，只在按下后的一小段音频里挑选音高；识别出调性后会优先从调内候选音中选择，减少滑音、尾音和气声造成的错音。新版会额外清理短过渡音，避免换音瞬间被切成零碎错音。

Space 手动分音支持“音头跳过 ms”和“采样窗口 ms”。系统会在按下 Space 后先略过一小段音头，再从偏中后段的稳定区域选择主音，减少咬字、辅音、滑入音造成的音高偏差。默认音头跳过为 80ms。

Space 模式会尽量保持“一个有效 Space 标记对应一个音符”。如果某个标记窗口里没有足够稳定的音高，系统会用延展窗口、邻近 Basic Pitch 候选或前后音生成一个低置信度占位音，方便在钢琴卷帘里手动校正，而不是直接丢掉这个音符块。

针对真实哼唱样本做了低置信度调性保护：自动识别出的调性如果不够确定，不会再强行把所有音符吸到调内。即使调性可信，也只会优先修正短、弱、像滑音碎片的离调音，稳定的变化音会保留；这能避免流行歌里常见的借用音、半音经过音被误删。八度修正也改成整句动态规划，不再只看相邻两个音贪心修正。

八度修正会额外检测“双向八度孤岛”：当一小段音突然高八度或低八度，而前后旋律仍在同一声区时，只有在移动 12 个半音后明显更连续的情况下才会修正，减少泛音误判或二分频误判造成的突然升/降八度。

新版不会把所有大跨度都当成错误：强拍上的音、时值较长的音、音量更明确的音，以及连续进入新声区的一串音，会被视为可能的真实大跳并保留。八度修正只优先处理短、弱、前后夹在同一声区里的突发八度孤岛。

八度判断现在优先相信原音频证据：帧级音高候选会保留分数和八度歧义，转成音符后会记录原始音频是否支持当前八度。只有当原音频八度证据不可靠时，才允许旋律连续性规则介入修正。

默认流程现在是先自由识别音高，再根据整段旋律显示“推测调性”。自动推测出的调性不会写入当前调性，也不会直接修改 MIDI 或提前生成调性和弦；需要时可以在识别修音页点击“按调性修正”，并可用“撤销调性修正”回到自由识别结果。如果录音前手动选择了固定调性，则会按用户选择的调性处理。

“按调性修正”现在是强制调内修正：所有调外音都会改成调内音。修正时会优先读取该音符从原始音频得到的 `pYIN` 稳定音高，再吸附到离这个原始音高最近的调内音；只有没有原始音频音高时，才退回当前 MIDI 音高判断。

录音前可以在“调性”里选择固定调性；默认是“自动识别”。选择 C major、G major、A minor 等调性时会立即播放该调主音作为基准音。生成 MIDI 时会按所选调性的调内音识别；选择“自动识别”则不锁定调性。

开启“智能配和弦”后，会根据识别出的调性和旋律自动生成和弦轨。当前算法升级为混合式动态规划：借鉴 AutoHarmonizer 的逐帧/密度控制思路、AccoMontage2 的 micro / meso / macro 三层目标，以及 BLSTM 类方法的前后文平滑思想。它会为每个候选片段的调内和弦、常见借用和弦、副属和弦、七和弦、sus4、add9 打分，再用动态规划选择整体最顺的和弦路线；强拍、长音、句尾音权重更高，弱拍经过音/邻接音处罚更轻。新版会额外识别琶音型骨干音，并要求借用/副属/扩展和弦具备明确的旋律证据；副属和弦如果能解决到目标和弦会加分，不能合理解决会扣分。和弦粒度支持智能自适应、一小节、半小节；智能自适应会根据伴奏密度决定内部候选帧，并自动合并连续相同和弦。算法还会在短语窗口内尝试 I-vi-IV-V、I-V-vi-IV、ii-V-I 等常见模板，并内置 4536251、1645、6415、1564、1625、3625 等中文数字和弦套路；只有模板路线的局部旋律评分足够时才套用。伴奏风格也会影响和弦复杂度，抒情更偏 maj7/add9，流行更偏稳定三和弦和属七。

开启“自动编曲”后，系统会把 Smart Chords 和弦轨进一步转换成钢琴伴奏轨。当前支持“流行节奏”“分解和弦”“抒情铺底”三种风格，以及简洁、标准、丰富三档密度。网页试听会优先播放自动伴奏；导出的 MIDI 会包含旋律轨、Smart Chords 和弦轨和 Auto Arrangement 自动编曲轨，方便导入编曲软件继续拆分编辑。

生成后可以点击和弦 chip，使用下拉框手动修改单个和弦；修改会立刻刷新网页试听和下载的 MIDI。

如果录音时开启节拍器，系统会记录节拍器第一拍相对录音起点的偏移。旋律量化、和弦生成、网页试听和导出 MIDI 的和弦轨都会对齐到同一套 BPM/拍号网格。

导出 MIDI 和网页 MIDI 试听会使用“音乐时间轴”：节拍器第一拍会对齐到 0 秒，录音启动前的极短 lead-in 不会进入 MIDI，因此旋律轨和 Smart Chords 和弦轨会从同一个第一拍开始。

点击“试听 MIDI”可以直接在网页里听导出的 MIDI 效果。钢琴音色来自本地 `piano.js` 音源模块，不需要联网下载 SoundFont。新版使用本地生成的采样式钢琴波形缓存，包含多弦轻微失谐、非整数泛音、击弦瞬态、音板共鸣、释放噪声、合唱宽度和轻混响，体积仍保持在很小的网页包内。

如果本地 Basic Pitch 已就绪，“生成 MIDI”会优先使用它识别音高，再把结果交给网页里的节拍对齐、智能配和弦、手动改音符、钢琴试听和最终 MIDI 导出流程。也可以用“导入外部 MIDI”载入其他工具导出的 `.mid`，继续使用本工作台整理。

自研识别引擎已升级为多候选音高追踪：每一帧会保留多个可能基频，再用动态规划选择整段最连贯的音高线。新版会把半周期、双周期一起作为谐波/子谐波证据打分，降低把泛音当主音、或把基频二分频的概率。后处理会折叠颤音/滑音造成的半音碎片，修正常见八度跳，并让 Space 分音模式优先相信当前片段的稳定主音，减少被上一音拖偏的情况。

生成 MIDI 后可以在钢琴卷帘里点选音符并手动修改音高、起点拍数、长度拍数和力度。点选或修改音符块时会立刻用本地钢琴音源试听该音符，音高编辑支持升/降半音和升/降八度，也支持按当前量化网格前移、后移、缩短和延长。可以选择 `1/4`、`1/8`、`1/16`、`1/32` 音符网格后点击“按节拍量化”；Space 模式会优先用每次按下 Space 的时间作为音头依据，再吸附到最近网格。钢琴卷帘会显示小节线、拍线、量化细分线和 Space 音头虚线，试听和下载会立即使用修改后的结果。

外置声卡或 USB 麦克风可以在“麦克风输入”里手动选择；如果插入设备后没有出现，点“刷新”后再开始录音。录音请求会先尝试低延迟、关闭浏览器自动处理的模式，如果声卡驱动不接受这些参数，会自动降级为更兼容的输入模式。对双通道声卡输入，会自动取当前信号更强的一路，避免麦克风插在 2 通道时被空通道稀释。

使用 Voicemeeter 时，网页里的“麦克风输入”通常要选 `VoiceMeeter Output` 或 `VoiceMeeter Aux Output` 这类录音端，不是播放端；Voicemeeter 里真实麦克风通道需要推到 `B1`，右上角 `A1` 选择真实声卡。

## 运行

```powershell
.\web\serve.ps1
```

然后打开：

```text
http://localhost:8765/
```

浏览器麦克风录音需要 `localhost`，不建议直接双击 `index.html`。

## 安装本地引擎

```powershell
.\web\install_engine.ps1
```

当前脚本安装的是 Basic Pitch 的 ONNX Runtime 路线，适合 Windows 本地服务使用。

## 打包

最终网页压缩包会生成到：

```text
dist\HumPitchWeb.zip
```
