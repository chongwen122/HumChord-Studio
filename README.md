# HumChord Studio

HumChord Studio 是一个本地运行的哼唱转 MIDI 工作台。它可以录制或导入哼唱旋律，把音频识别成可编辑的 MIDI 音符，并支持节拍量化、按调性修正、钢琴卷帘手动编辑、智能配和弦、自动钢琴编曲、网页试听和 MIDI 导出。

当前识别链路使用 Spotify 开源模型 Basic Pitch，通过 ONNX Runtime 在本机运行；随后再用单声部 pYIN 音高轨迹对哼唱音高做二次校准。所有音频都在本机处理，不会上传到云端。

## 主要功能

- 在浏览器里录制哼唱，或导入已有音频。
- 支持自定义 BPM 和拍号的节拍器。
- 默认开启 Space 手动分音：录音时每唱一个字或一个音按一次 `Space`。
- 使用 Basic Pitch + pYIN 把哼唱转换成 MIDI。
- 在钢琴卷帘里编辑 MIDI 音符。
- 点击或修改音符时自动试听该音符。
- 支持升降半音、升降八度。
- 支持 `1/4`、`1/8`、`1/16`、`1/32` 网格量化。
- 自动推测调性，也可以手动选择调性。
- 支持把所有调外音强制修正到当前调内音。
- 自动生成和弦伴奏，并支持手动修改单个和弦。
- 和弦算法支持智能自适应换和弦、常用七和弦、sus、add9。
- 根据和弦自动生成钢琴伴奏编曲，支持流行节奏、分解和弦、抒情铺底。
- 使用本地钢琴音源试听旋律、和弦和自动伴奏。
- 导出最终 MIDI 文件。

## 运行环境

目前主要在 Windows 上测试。

你需要：

- Python 3.12 或相近版本的 Python 3
- PowerShell
- Chrome / Edge 等现代浏览器
- 第一次安装依赖时需要联网
- 麦克风、声卡输入，或已有音频文件

浏览器录音需要通过 `localhost` 打开页面。不要直接双击 `web/index.html`，否则麦克风权限和本地识别接口可能无法正常工作。

## 下载项目

使用 Git 克隆：

```powershell
git clone https://github.com/chongwen122/HumChord-Studio.git
cd HumChord-Studio
```

也可以在 GitHub 页面点击 `Code` -> `Download ZIP`，下载后解压，然后在解压后的文件夹里打开 PowerShell。

## 第一次安装

第一次使用前，先安装本地识别引擎：

```powershell
.\web\install_engine.ps1
```

这个脚本会创建或复用 `.venv`，并安装：

- Basic Pitch
- ONNX Runtime
- librosa
- pretty-midi
- scipy / scikit-learn
- 其他音频处理依赖

脚本会安装 ONNX Runtime 版 Basic Pitch，避免在 Python 3.12 下拉取不兼容的旧 TensorFlow。这是正常设计，不是缺依赖。

如果 PowerShell 阻止脚本运行，可以先执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新运行：

```powershell
.\web\install_engine.ps1
```

## 启动应用

运行：

```powershell
.\web\serve.ps1
```

然后在浏览器打开：

```text
http://localhost:8765/
```

使用过程中请保持 PowerShell 窗口打开。想停止应用时，在 PowerShell 窗口按 `Ctrl+C`。

如果本地引擎可用，页面会显示 Basic Pitch / ONNX 状态。如果本地引擎不可用，网页会退回内置浏览器识别引擎，但准确率通常会低一些。

## 基本使用流程

1. 打开 `http://localhost:8765/`。
2. 设置 BPM 和拍号。
3. 调性可以保持 `自动识别`，也可以在录音前手动选择固定调性。
4. 点击开始录音，或导入已有音频。
5. 点击 `生成 MIDI`。
6. 在 `识别修音` 页查看钢琴卷帘。
7. 如果有错音，可以手动修改音高、起点、长度和力度。
8. 如果节奏需要对齐，点击 `按节拍量化`。
9. 如果想把所有调外音修到调内，点击 `按调性修正`。
10. 在 `和弦编配` 页生成或修改和弦。
11. 在 `试听导出` 页试听并下载 MIDI。

## 录音建议

为了提高识别准确率：

- 尽量只唱一条清晰旋律线。
- 录音时不要同时播放伴奏或原曲。
- 麦克风离嘴近一些，但不要爆音。
- 如果在意节奏，请开启节拍器跟着唱。
- 如果是带歌词、按字唱、或每个音比较分明，建议开启 Space 手动分音。
- 按下 Space 后，尽量把该音保持一小段时间，方便系统分析稳定音高。
- 尽量减少特别重的气声、喷麦和滑入音。

## Space 手动分音

Space 手动分音默认开启。

录音时，每唱一个字、一个音，按一次 `Space`。系统会尽量保持一个有效 Space 标记对应一个 MIDI 音符。

如果某个 Space 标记窗口里没有检测到足够稳定的音高，系统不会直接丢掉这个标记，而是生成一个低置信度占位音。你可以之后在钢琴卷帘里手动修改它。

如果两次 Space 间隔小于约 `80ms`，系统会认为这是误触或连击抖动，并自动过滤。

相关参数：

- `音头跳过 ms`：按下 Space 后跳过一小段音头，默认 `80ms`。
- `采样窗口 ms`：跳过音头后用于分析稳定音高的窗口，默认 `420ms`。

## 调性识别和按调性修正

系统会先自由识别音高，然后根据整段旋律推测调性。

调性判断会更重视：

- 长音
- 句尾音
- pYIN 判断稳定的音高
- 置信度较高的音

短促的滑音、过渡音、噪声碎片对调性判断的影响会更小。

`按调性修正` 会强制把所有调外音修正到当前调内音。它不是只修短音或弱音。

修正时会优先查看该音符从原始音频得到的 pYIN 稳定音高，然后吸附到离这个原始音高最近的调内 MIDI 音。如果没有 pYIN 音高，才会退回使用当前 MIDI 音高。

如果自动调性判断不对，可以先在调性下拉框里手动选择正确调性，再点击 `按调性修正`。

如果修正后不满意，可以点击 `撤销调性修正`。

## 节拍量化

`按节拍量化` 会把每个音符的音头吸附到最近的网格位置。开启 Space 手动分音时，系统会优先使用你按下 `Space` 的时间作为音头依据，而不是用识别算法切出来的音符起点；如果多个标记被吸到同一条线附近，会保持原来的演唱顺序，把后面的音放到下一条可用网格线上，避免音符重叠或丢失。

可选网格：

- `1/4`
- `1/8`
- `1/16`
- `1/32`

量化基于当前 BPM、拍号和录音时的节拍器偏移。如果录音时跟着内置节拍器唱，量化效果会更稳定。钢琴卷帘会显示小节线、拍线、当前量化细分线和 Space 音头虚线，方便对照节奏。

## MIDI 手动编辑

在钢琴卷帘里可以：

- 点击音符块选中并试听。
- 修改音高。
- 修改起点拍数。
- 修改长度拍数。
- 修改力度。
- 升半音 / 降半音。
- 升八度 / 降八度。
- 按当前量化网格前移 / 后移一格。
- 按当前量化网格缩短 / 延长一格。
- 删除错误音符。

每次修改后，试听和下载的 MIDI 都会自动刷新。

## 智能配和弦

`智能配和弦` 会根据识别出的旋律和调性生成和弦轨。

当前和弦算法是一个混合式可解释版本，借鉴了 AutoHarmonizer 的逐帧/密度控制思路、AccoMontage2 的 micro / meso / macro 三层目标，以及 BLSTM 类方法的前后文平滑思想。它不会直接依赖旧 Keras 模型，而是在本地 Worker 里用动态规划实时生成。

算法会综合考虑：

- 旋律音和和弦音的匹配度
- 强拍音和长音
- 句尾音
- 简单声部连接
- 常见借用和弦和副属和弦候选
- 常用七和弦、sus4、add9 的旋律证据
- 常见流行和声模板，如 I-vi-IV-V、ii-V-I、V-I
- 中文数字和弦套路，如 4536251、1645、6415、1564、1625、3625
- 短语窗口里的模板匹配，只有局部旋律评分足够时才套用
- 伴奏风格对和弦复杂度的影响，抒情更偏色彩和弦，流行更偏稳定和弦

可以选择和弦变化密度：

- 智能自适应
- 一小节一个和弦
- 半小节一个和弦

`智能自适应` 会根据伴奏密度自动选择内部候选帧：简洁时更接近一小节，标准时更接近半小节，丰富时允许更频繁的和弦变化。相邻相同和弦会自动合并，避免生成过碎的和弦块。

内置数字套路库包括：`4536251`、`1645`、`6415`、`1564`、`1451`、`251`、`451`、`456`、`1625`、`3625`、`6251`、`2516`、`4361`、`4536`、`6451`、`4561`。这些套路会作为参考路线参与评分，不会无条件覆盖旋律本身。

生成后，可以点击和弦 chip，然后在下拉框里手动修改单个和弦。

## 自动编曲

`自动编曲` 会把当前和弦转换成一条独立的钢琴伴奏轨。网页试听时会优先播放自动伴奏，不再只播放长音和弦；导出的 MIDI 会包含旋律轨、Smart Chords 和弦轨、Auto Arrangement 自动编曲轨，方便在 DAW 里继续拆分编辑。

可选伴奏风格：

- `流行节奏`：低音加切分和弦，适合流行歌副歌或主歌铺底。
- `分解和弦`：按拍子循环分解和弦，适合抒情或慢歌。
- `抒情铺底`：低音加轻微滚奏和弦，听感更柔和。

可选伴奏密度：

- `简洁`
- `标准`
- `丰富`

自动编曲基于当前 BPM、拍号、节拍偏移和手动修改后的和弦。修改单个和弦或切换伴奏风格后，试听和下载的 MIDI 会自动刷新。

## MIDI 试听和导出

项目内置一个小体量本地钢琴音源模块：`web/piano.js`。它不需要联网下载 SoundFont。

点击 `试听 MIDI` 可以听旋律和自动伴奏。点击 `下载 MIDI` 可以导出最终 `.mid` 文件，再放进 DAW 或编曲软件继续处理。

## 外置声卡和 Voicemeeter

如果使用 USB 麦克风或外置声卡：

- 在 `麦克风输入` 下拉框里选择正确输入。
- 插入设备后如果没出现，点击 `刷新`。
- 确认浏览器已经允许麦克风权限。

如果使用 Voicemeeter：

- 网页里的麦克风输入通常选择 `VoiceMeeter Output` 或 `VoiceMeeter Aux Output`。
- Voicemeeter 里真实麦克风通道需要推到 `B1`。
- `A1` 通常是物理监听输出，不是浏览器录音输入。

如果声卡只开了 ASIO 输入，浏览器通常不能直接读取 ASIO。可以通过 Voicemeeter 或声卡的 WDM/MME 输入通道转给浏览器。

## 常见问题

### 页面能打开，但不能录音

- 确认地址是 `http://localhost:8765/`，不是本地文件路径。
- 检查浏览器麦克风权限。
- 点击输入设备旁边的 `刷新`。
- 关闭可能占用麦克风的软件。
- 先试试系统默认输入。

### 本地识别引擎不可用

重新运行：

```powershell
.\web\install_engine.ps1
.\web\serve.ps1
```

如果依赖安装失败，先确认 Python 可用：

```powershell
python --version
```

### 识别出来的音太多

- 开启 Space 手动分音。
- 提高 `最短 ms`。
- 使用 `按节拍量化`。
- 在钢琴卷帘里删除明显短碎音。

### 识别出来的音太少

- 开启 Space 手动分音，并每个字/音按一次 Space。
- 降低 `最短 ms`。
- 每个音唱得稍微长一点。
- 提高麦克风输入音量。

### 调性判断不准

- 在调性下拉框里手动选择正确调性。
- 再点击 `按调性修正`。
- 如果歌曲里有很多经过音、转调或借用音，不建议过早强制调性修正。

### MIDI 没声音

- 先点击一下页面，让浏览器允许音频播放。
- 再点击 `试听 MIDI`。
- 检查系统输出音量。

## 开源模型说明

HumChord Studio 使用 Spotify 的 Basic Pitch 作为本地识别模型。Basic Pitch 是 Apache-2.0 许可证的开源项目。

项目地址：

```text
https://github.com/spotify/basic-pitch
```

本项目不包含 Melodyne，不绕过 Melodyne 授权，也不依赖 Melodyne。
